About

Alexandria is a text-adventure engine that runs in the browser. It uses a simnple text parser to interpret commands and it reads from specially formatted JSON files. This guide will teach you how a story JSON file is organized and how to write your own.


In this guide, I've included examples of plain JSON for perusal as well as many small clarifications. If you see any text among the paragraphs that is colored red, try hovering over it to read the note.

What is JSON?

JSON stands for JavaScript object notation. It's easy to read and write, even if you're not already completely familiar with JavaScript literal notation. Most of it is composed of key / value pairs, but keys can also hold sets of key / value pairs themselves.


If you'd like to get a good handle on the basics of how JSON is used, see JSON: What It Is, How It Works, & How to Use It. To get an idea of how it's formatted, see examples here. If you'd like help linting (debugging) your JSON, you can google the many formatters and validators available online (I prefer this one, myself.)

Anatomy of a story file

This is an example of a simple one-scene story file:


"scene0": {
  "html": "This is where you put the description of a scene, formatted in
           HTML.",

  "events": {
    "event0": "This is an event."
  },

  "moves": {
    "east": "scene1" },

  "objects": {
    "torch": {
    "name": "torch",
    "description": "Some oil-soaked cloth wrapped around a stick. You could
                    light it if you had a source of fire.",
    "is_unique": "true",
    "is_usable": "true",
    "is_takeable": "true",
    "take_message": "You grab the torch and put it in your pack.",
    "on_use": "The torch burns away the cobwebs, revealing a passage" }
  },

  "tag_init": {
    "tag-1": "true", },

  "npc0": {
    "name": "The name you put here is how players will look at or talk to the
             npc. It need not be the NPC's actual name, but should instead be
             one-word descriptor like tiger or woman.",
    "description": "NPC descriptions go here.",
    "dialog": {
      "0": "This is where npc dialog goes."
    }
  }
}

Scene basics:

At its heart, all a scene really needs to function are a description and a list of possible moves. Scenes are declared in the story file as "scene + a number" (e.g. "scene0, scene1, scene10.") Descriptions should be written as properly tagged HTML, e.g. <p>, <h1>, <b>, etc.


When writing scene descriptions, it's good practice to highlight possible command targets for the player. This is done with bold tags e.g. "There is an open door to the <b>east<b>" or "a strange <b>woman<b> sits on a chair on the balcony." This practice ensures that a player will understand their options in a scene. On the other hand, not highlighting an important aspect of a scene can be used as part of a puzzle.

In this room is a stone door locked by a strange mechanism. Tiles with depictions of animals surround the door on all sides. There is an inscription upon the arch of the door.
>> look inscription
You read the inscription:
"I walk on four legs and often wear shoes? What am I?"
>> use horse tile
It seems your answer was correct. The door opens slowly.

Beware of using this technique as it can easily leave your player lost if not used skillfully.


Moves are very simple to define. Under the moves property of a scene, a key / value pair must be declared for each possible exit. The key string should be the same as the noun component that you expect a player to enter with the move command. The value of that key should be equal to whichever scene the player should be moved to. For example:

Moves: {
  "KEY"  : "VALUE",
  "east" : "scene3",
  "south": "scene4",
  "cave" : "scene2",
  "well" : "scene10"
}

At this time, move keys must be one word long.

Events:

Events can serve many purposes, and are triggered by many different things:

  • Looking at an object opens an event with a description of the object.
  • Successfully using objects can cause events to occur.

Upcoming event triggers (not yet implemented):

  • Events can be triggered by moving to an area.
  • Events can be triggered by talking to NPCs.
  • Some events can only occur if specific tags are active.
